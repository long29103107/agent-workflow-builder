Agent Workflow Builder
Source of truth: AgentWorkflow.Core

Nền tảng điều phối agent cho việc điều tra task và lập kế hoạch thực thi

Dự án này gom UI, API, CLI và MCP vào cùng một lõi dùng chung. Người dùng hoặc hệ thống bên ngoài đưa task vào, Lead Agent đọc context, gọi các subagent, tra memory, đọc repository, rồi tạo summary và execution plan.

React UI ASP.NET Core API CLI adapter MCP stdio adapter OpenAI SDK reasoning

Dự án này dùng để làm gì?

Mục tiêu là tạo một control plane cho agent workflow: nhận task, thu thập bối cảnh từ nhiều nguồn, điều phối subagent chuyên trách, và xuất ra kế hoạch có thể review trước khi triển khai thật.

Điều tra task có cấu trúc

Lead Agent gom Jira/Notion context, repository context, memory và graph links để tránh lập kế hoạch dựa trên thông tin rời rạc.

Một lõi dùng chung

`AgentWorkflow.Core` giữ domain model, interfaces, orchestration và mock integrations. API, CLI, MCP chỉ là adapter mỏng.

Sẵn sàng thay mock bằng thật

Jira, Notion, repository reader, memory stores và OpenAI reasoning đều nằm sau interface, nên có thể thay bằng provider thật theo từng phase.

Workflow tổng thể

Các entrypoint khác nhau đều đi vào `AgentWorkflow.Core`, sau đó Core điều phối Lead Agent và subagents để tạo kết quả nhất quán cho UI, API, CLI hoặc MCP caller.

Các cách gọi vào hệ thống

Bốn adapter phục vụ bốn nhóm người dùng khác nhau nhưng không nhân đôi logic. Tất cả cùng gọi vào Core.

UI

Cho người dùng kéo task vào lane điều tra, xem timeline agent và đọc execution plan.

API

Cung cấp endpoint HTTP như `/api/tasks` và `/api/workflows/investigate` cho frontend hoặc service khác.

CLI

Cho automation hoặc developer chạy workflow từ terminal và nhận JSON output ổn định.

MCP

Cho tool host gọi workflow qua stdio JSON, chuẩn bị đường mở rộng sang MCP server thật.

Thành phần chính trong Core

Core giữ các interface và implementation mock đầu tiên, để sau này thay từng mảnh bằng provider thật mà không làm đổi API/CLI/MCP/UI.

Lead Agent

  • Nhận task từ request.
  • Đọc Jira/Notion context, repository context và memory.
  • Gọi các subagent chuyên trách.
  • Tổng hợp kết quả thành plan.

Subagents

  • Repository Investigator Agent.
  • Jira/Notion Context Agent.
  • Memory Research Agent.
  • Planning Agent.

Memory và repo context

  • Mock vector memory chuẩn bị cho Qdrant.
  • Mock graph relationships chuẩn bị cho Neo4j.
  • Local repository reader để lấy file và technology signals.

OpenAI reasoning

  • Dùng OpenAI .NET SDK khi có `OPENAI_API_KEY`.
  • Fallback deterministic khi chưa cấu hình key.
  • Ẩn sau `IAgentReasoningService` để thay model/provider dễ hơn.

Ý nghĩa kiến trúc

Điểm quan trọng nhất là dependency đi vào trong: UI, API, CLI và MCP không tự xử lý agent logic. Chúng gọi `AgentWorkflow.Core`, nơi giữ workflow thật. Nhờ vậy hệ thống có thể mở rộng thêm real Jira, real Notion, Qdrant, Neo4j, GitHub/GitLab hoặc OpenAI model khác mà không làm mỗi entrypoint phát triển lệch nhau.
