# Project Context

This folder stores durable context that Codex must read before planning new work.

## Files

- `project-context.md`: current architecture, stack, source-of-truth rules, and operating assumptions.
- `task-workflow.md`: how to choose a phase/task ID, implement work, and write memory logs.

## Rule

Before starting a new implementation task, read:

1. `AGENTS.md`
2. `.codex/context/project-context.md`
3. `.codex/phases/README.md`
4. Relevant `.codex/phases/*.md`
5. Existing `.codex/memories/tasks/*.md` for related task IDs
